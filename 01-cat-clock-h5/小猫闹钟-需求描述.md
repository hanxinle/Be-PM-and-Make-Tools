# 小猫闹钟 - 产品需求文档

## 1. 概述

一个基于单 HTML 文件的浏览器端闹钟应用。用户可添加多个一次性闹钟，每个闹钟支持自定义时间和闹铃音频。闹钟触发后保留在列表中并标记为已响铃。

---

## 2. 页面结构与视觉设计

### 2.1 整体布局

- 页面标题为 `小猫闹钟`，加载 `cat_icon.png` 作为 favicon。
- 背景：`linear-gradient(135deg, #1a1a2e 0%, #16213e 100%)`。
- 页面内容居中，使用 flex 布局，最小高度 `100vh`，四周留 `20px` 内边距。
- 字体栈：`"Microsoft YaHei", "Segoe UI", sans-serif`。
- 全局文字颜色：`#eee`。

### 2.2 主容器（`.container`）

- 居中卡片，最大宽度 `500px`，宽度占满父容器。
- 背景：`rgba(255, 255, 255, 0.05)`。
- 圆角：`20px`，内边距：`40px`。
- 毛玻璃效果：`backdrop-filter: blur(10px)`。
- 边框：`1px solid rgba(255, 255, 255, 0.1)`。

### 2.3 标题

- `h1`：居中，下边距 `30px`，字号 `2em`，颜色 `#00d4ff`。

### 2.4 当前时间（`.current-time`）

- 居中大号显示，字号 `3em`，加粗，白色，带文字阴影 `0 0 20px rgba(0, 212, 255, 0.5)`。
- 每秒更新，格式为 `HH:MM:SS`（24 小时制）。

## 3. 控制区域

### 3.1 编辑提示（`.edit-hint`）

- 默认隐藏（`display: none`）。
- 进入编辑模式时显示文字：`"正在编辑闹钟，修改后点击保存"`。
- 文字颜色：`#00d4ff`，字号 `0.9em`，居中。

### 3.2 时间选择器（`.control-panel`）

- 横向 flex 布局，gap `10px`，垂直居中对齐。

#### 3.2.1 时间输入区（`.time-picker`）

- 内部 flex 布局，gap `5px`，占满剩余宽度。
- 背景：`rgba(255, 255, 255, 0.1)`，边框：`1px solid rgba(255, 255, 255, 0.2)`，圆角 `10px`，内边距 `8px`。

包含两个输入框和一个分隔符：

- **小时输入框**（`id="hourUnit"`）：
  - 类型为 `text`，`inputmode="numeric"`，最大长度 `2`，初始值 `"12"`。
  - 样式类 `.time-unit`：flex 占满、居中、圆角 `8px`、字号 `1.6em`、加粗、白色、背景 `rgba(255,255,255,0.05)`、无边框、无外框线、无数字微调按钮（`-webkit-appearance: none`）。
  - 鼠标悬停时背景变为 `rgba(0, 212, 255, 0.2)`。
  - 聚焦时边框变为 `#00d4ff`，背景 `rgba(0, 212, 255, 0.15)`，光标变为文本输入态，聚焦时自动全选文本（`onfocus="this.select()"`）。
  - **滚轮事件**：向上滚动小时 `+1`（模 24），向下滚动 `-1`（模 24）。
  - **键盘事件**：
    - `Enter`：失焦确认。
    - `ArrowUp` / `ArrowDown`：分别增减 1（模 24）。
  - **输入过滤**：只允许数字，最多 2 位（正则替换 `/\D/g`）。
  - **失焦校验**（`onblur`）：若为空、非数字、小于 0 或大于 23，弹出 `alert('小时必须是 0 到 23 之间的整数。')`，恢复为当前有效值，并重新聚焦全选。

- **分隔符**：`:`，字号 `1.6em`，加粗，颜色 `#00d4ff`。

- **分钟输入框**（`id="minuteUnit"`）：
  - 与小时输入框结构完全一致，初始值 `"00"`。
  - 滚轮和键盘箭头操作分钟（模 60）。
  - 失焦校验范围 0~59，错误提示：`"分钟必须是 0 到 59 之间的整数。"`。

#### 3.2.2 按钮

- **主按钮**（`id="actionBtn"`）：
  - 默认文字：`"设置闹钟"`。
  - 编辑模式下文字变为：`"保存修改"`。
  - 样式：内边距 `12px 24px`，圆角 `10px`，无边框，渐变背景 `#00d4ff -> #0099cc`，白色文字。
  - 悬停：上移 `2px` + 阴影 `0 5px 20px rgba(0, 212, 255, 0.4)`。
  - 点击时恢复上移。

- **取消按钮**（`id="cancelBtn"`）：
  - 文字：`"取消"`。
  - 默认隐藏（`display: none`），编辑模式显示为 `inline-block`。
  - 样式类 `.secondary`：渐变背景 `#666 -> #444`。

### 3.3 内容输入框（`id="contentInput"`）

- 占位符：`"闹钟内容（可选）"`。
- 全宽，圆角 `10px`，边框和背景同 `.time-picker`，白色文字，字号 `1em`。
- 占位符颜色：`#888`。
- 下边距 `10px`。

### 3.4 闹铃选择器（`.file-picker`）

- flex 横向布局，垂直居中，gap `10px`，下边距 `10px`。
- 包含：
  - `<label for="audioInput">选择闹铃</label>`：样式为可点击按钮（背景 `rgba(255,255,255,0.1)`、边框 `1px solid rgba(255,255,255,0.2)`、圆角 `8px`、内边距 `8px 16px`、颜色 `#ccc`、字号 `0.9em`）。悬停时背景变 `rgba(0, 212, 255, 0.2)`，边框变 `#00d4ff`。
  - `<input type="file" id="audioInput" accept=".mp3,.wav,.ogg,audio/*">`：隐藏（`display: none`）。
  - 文件名显示区（`id="fileName"`）：弹性占满剩余空间，颜色 `#aaa`，字号 `0.9em`，单行省略。默认文字：`"使用默认闹铃 (bell3.mp3)"`。
- 用户选择文件后：
  - 使用 `URL.createObjectURL(file)` 创建临时音频 URL。
  - 文件名显示区更新为所选文件名。

- 默认隐藏（`display: none`），触发时添加 `.active`（`display: flex`）。
- 固定定位全屏遮罩，背景 `rgba(0, 0, 0, 0.9)`，z-index `1000`。
- 内容垂直居中排列：
  1. **GIF 动画**：`bk.gif`，尺寸 `400px x 300px`，圆角 `15px`，下边距 `20px`。
  2. **闹钟文字**（`id="alarmText"`）：`1.8em`，白色，下边距 `20px`。格式：`闹钟时间：HH:MM`，若有内容则追加 ` - {内容}`。
  3. **倒计时**（`id="countdown"`）：颜色 `#aaa`，下边距 `20px`。初始文字：`"20 秒后自动关闭"`，每秒递减。

- 弹窗显示后启动 20 秒定时器，到时自动调用停止逻辑。

## 7. 数据模型

### 7.1 闹钟对象（Alarm）

```
{
  time: string,       // "HH:MM" 格式
  content: string,    // 闹钟内容，可能为空
  audioUrl: string,   // 自定义闹铃的 blob URL，无则为 null
  audioName: string,  // 自定义闹铃文件名，无则为 null
  triggered: boolean  // 是否已响铃，默认 false
}
```

### 7.2 全局状态

- `alarms`: Alarm[] -- 闹钟列表。
- `selectedAlarmIndex`: number -- 当前选中项索引，`-1` 表示未选中。
- `editIndex`: number -- 当前编辑项索引，`-1` 表示非编辑模式。
- `hour`: number -- 时间选择器当前小时，默认 `12`。
- `minute`: number -- 时间选择器当前分钟，默认 `0`。
- `audioElement`: Audio -- 默认音频实例（`bell3.mp3`）。
- `currentAlarmAudio`: Audio -- 当前正在播放的自定义闹铃实例。
- `selectedAudioFile`: File -- 当前选中的文件对象。
- `selectedAudioUrl`: string -- 当前选中的文件 blob URL。
- `isAudioUnlocked`: boolean -- 音频是否已解锁。
- `alarmModalTimer`: timeout ID -- 弹窗自动关闭定时器。
- `countdownTimer`: interval ID -- 倒计时更新定时器。

## 8. 核心交互逻辑

### 8.1 初始化

- 页面加载时：
  - 初始化时间显示为 `12:00`。
  - 渲染闹钟列表（空状态）。
  - 立即显示当前时间，然后每秒更新。
- 监听首次点击，调用音频解锁逻辑。

### 8.2 添加闹钟流程

1. 用户点击主按钮（`"设置闹钟"`）。
2. 先执行 `commitHour()` 和 `commitMinute()` 确保时间值有效。
3. 若非编辑模式，进入添加流程：
   - 获取 `HH:MM` 和内容（去首尾空格）。
   - **去重校验**：遍历 `alarms`，若有其他项（`idx !== editIndex`）的 `time` 和 `content` 完全相同，弹出 `alert('该时间和内容的闹钟已存在！')`，中断。
   - 创建 Alarm 对象，`triggered: false`。
   - 推入 `alarms` 数组。
   - 调用 `resetInputs()` 清空输入区。
   - 重新渲染列表。

### 8.3 编辑闹钟流程

1. 用户点击列表中某一项。
2. 该项设为选中 + 编辑状态：
   - `selectedAlarmIndex = index`，`editIndex = index`。
   - 将该闹钟的时间回填到选择器。
   - 内容回填到输入框。
   - 若有自定义闹铃，文件名显示区显示其文件名（blob URL 回填到 `selectedAudioUrl`）；无则显示默认提示。
   - 文件输入框本身清空（安全限制无法回填）。
   - 主按钮文字变为 `"保存修改"`。
   - 取消按钮显示。
   - 编辑提示显示。
   - 重新渲染列表（高亮选中项）。
3. 用户修改后点击 `"保存修改"`：
   - 同样执行 `commitHour()` 和 `commitMinute()`。
   - 去重校验（排除当前编辑项）。
   - 更新 `alarms[editIndex]`，**必须将 `triggered` 重置为 `false`**。
   - 调用 `cancelEdit()` 退出编辑模式。
   - 重新渲染列表。

### 8.4 取消编辑（`cancelEdit`）

- `editIndex = -1`。
- 隐藏编辑提示。
- 主按钮文字恢复为 `"设置闹钟"`。
- 隐藏取消按钮。
- 调用 `resetInputs()`。
- 若有选中项，清除选中状态并重新渲染。

### 8.5 重置输入（`resetInputs`）

- 时间恢复为 `12:00`。
- 内容输入框清空。
- 文件输入框清空。
- `selectedAudioFile = null`，`selectedAudioUrl = null`。
- 文件名显示区恢复为 `"使用默认闹铃 (bell3.mp3)"`。

### 8.6 删除闹钟

- 检查 `selectedAlarmIndex`，`-1` 时提示 `alert('请先选择要删除的闹钟！')`。
- 从 `alarms` 中删除该项。
- 若删除的是正在编辑的项，调用 `cancelEdit()`。
- 若删除项在编辑项之前，`editIndex--`。
- `selectedAlarmIndex = -1`。
- 重新渲染列表。

### 8.7 闹钟检查（每秒执行）

- 获取当前时间，格式化为 `HH:MM`。
- 遍历 `alarms`，找到第一个 `time === 当前时间 && !triggered` 的项。
- 触发它：
  - 调用 `triggerAlarm(alarm)`。
  - 将该闹钟的 `triggered` 设为 `true`。
  - 重新渲染列表（更新样式为已触发）。
- 只触发一个，找到后即 `break`。

### 8.8 触发闹钟（`triggerAlarm`）

1. 显示弹窗（添加 `.active`）。
2. 设置弹窗文字。
3. 音频处理：
   - 若已有自定义音频在播放，先暂停重置。
   - 若该闹钟有自定义 `audioUrl` 且音频已解锁：新建 `Audio` 实例，循环播放。
   - 否则若默认音频已就绪：播放默认 `bell3.mp3`。
4. 启动倒计时显示（每秒更新，从 20 递减到 0）。
5. 启动 20 秒自动关闭定时器，到时调用 `stopAlarm()`。

### 8.9 停止闹钟（`stopAlarm`）

- 隐藏弹窗。
- 暂停并重置 `currentAlarmAudio`（若有）。
- 暂停并重置默认 `audioElement`。
- 清除自动关闭定时器和倒计时定时器。

## 9. 边界情况

- 同时间同内容的闹钟不允许重复添加（含编辑后的冲突检测）。
- 已触发的闹钟在同一天同一分钟不会再次触发（靠 `triggered` 字段控制）。
- 编辑已触发的闹钟会自动重置 `triggered` 为 `false`，使其可再次触发。
- 浏览器自动播放策略要求用户先与页面交互才能播放音频；首次点击前显示黄色提示条。
- 自定义闹铃通过 `URL.createObjectURL` 创建，仅在当前页面会话有效（刷新后失效，与闹钟列表不持久化的行为一致）。
- 时间输入框支持鼠标滚轮、键盘箭头、直接输入三种调节方式，均有范围校验。
- 弹窗显示期间，若另一闹钟时间到达，不会叠加触发（因为弹窗占用时 `checkAlarms` 仍在运行，但 `triggered` 标记阻止重复，且第一个触发后即 `break`）。
